Model Report

The Problem: Poor Indoor Air Quality

🏥
Health Impacts
Air pollution and poor indoor air quality lead to respiratory illnesses, allergies, and long-term diseases.
🏙️
Urban Challenges
Urban areas experience high pollution levels, but air quality monitoring devices remain expensive and inaccessible.
⏱️
Lack of Prediction
Existing solutions lack predictive insights, making it difficult to prevent air quality issues before they become hazardous.
💡
Solution Needed
Need for an affordable, AI-driven system to provide real-time air quality monitoring and proactive alerts.

Our Solution

A comprehensive IoT-based environmental monitoring solution that combines embedded sensor technology with cloud-based machine learning

💨
Low-cost Monitoring

Comprehensive Sensing
Measures temperature, humidity, pressure, CO₂ levels, and particulate matter (PM2.5 & PM10)

AI Integration
Predicts air quality trends and warns users in advance of potential issues
💾

Display Interface
LCD displays real-time sensor readings in an easy-to-understand format
🔔
Alert System

Database Design

Optimized schema for IoT sensor data storage

Database Schema

CREATE TABLE IF NOT EXISTS air_quality_data ( id INTEGER PRIMARY KEY AUTOINCREMENT, timestamp TEXT NOT NULL, device_id TEXT, co2 REAL, pm2_5 REAL, pm10 REAL, temperature REAL, humidity REAL, co2_category INTEGER, pm2_5_category INTEGER, pm10_category INTEGER, hour INTEGER, day_of_week INTEGER, is_weekend INTEGER, prediction INTEGER, probability REAL );
CREATE TABLE IF NOT EXISTS fire_detection_data ( id INTEGER PRIMARY KEY AUTOINCREMENT, timestamp TEXT NOT NULL, device_id TEXT, temperature REAL, humidity REAL, tvoc REAL, eco2 REAL, raw_h2 REAL, raw_ethanol REAL, pressure REAL, pm1_0 REAL, pm2_5 REAL, nc0_5 REAL, nc1_0 REAL, nc2_5 REAL, prediction INTEGER, probability REAL );

Troubleshooting

Common issues and their solutions

Hardware Issues

Issue Possible Causes Solutions
BME680 not detected
  • I2C address mismatch
  • Hardware connection issues
  • Verify I2C address (0x76 or 0x77)
  • Check wiring connections
ENS160 not responding
  • Incorrect I2C address
  • Sensor in sleep mode
  • Verify address (0x53)
  • Send wake-up command
LCD Display not working
  • I2C address incorrect
  • Contrast setting issue
  • Verify I2C address (typically 0x27)
  • Adjust contrast potentiometer
PMS5003 communication errors
  • Incorrect UART pins
  • Baud rate mismatch
  • Verify TX/RX connections
  • Confirm baud rate (9600)

Connectivity Issues

Issue Possible Causes Solutions
ESP8266 not responding
  • Incorrect UART pins
  • Baud rate mismatch
  • Verify TX/RX connections
  • Check baud rate (115200)
WiFi connection failures
  • Incorrect credentials
  • AP out of range
  • Verify SSID and password
  • Check signal strength
